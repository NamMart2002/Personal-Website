POM & Honey Web Application









Description


In this project, I worked in the same team as I did on the Java GUI project. We wanted to bring the system online in form the of a Web App, and we wanted to make a few improvements. We wanted to add a self-service portal for customers, improve accessibility, and add a few APIs.


The only thing that we could keep from our GUI was the PostgreSQL database that we designed; everything else had to be rebuilt from the ground up. Instead of using Java for both the front and back end, our new project was going to be built with HTML, CSS, JavaScript, and Express.js. We'd also go on to implement the Google Translate, Google OAuth, and Google Maps APIs in our Web App.





My Role



As in our Java GUI, I was placed in charge of our back end. This was definitely much more challenging for me than our GUI, as I had very little experience with JavaScript and absolutely none with Node or Express. I also implemented the Google Translate API in our project to make it more accessible to non-English speakers.





What I Learned


Although I wasn't super familiar with JavaScript going into this project, it being C based made it easy for me to pick up. Learning Express was significantly more challenging, but the more I used it the easier it became to use the framework. Eventually, I was able to make post requests, redirect pages, and handle database queries with ease. I spent most of my time on this project working on the back end, but I did end up also learning a decent amount of new tricks for HTML, CSS, and front end development in general from working with one of my group members
